Sobre los Tweets en el Paro Nacional

23 de Enero de 2020 en Twitter

Los últimos meses del año pasado presenciaron movilizaciones en algunos países del mundo, respondiendo a acciones gubernamentales de diversas índoles. En Ecuador, las protestas de octubre fueron detonadas por las medidas económicas anunciadas por la presidencia, con el propósito de cumplir una agenda acordada con el Fondo Monetario Internacional. El principal cambio consistía en terminar el subsidio de combustibles; como suele suceder con las medidas inmediatas, que no contemplan un cambio progresivo ni el desarrollo equitativo para todos, ciertos sectores -estudiantes, transportistas, agricultores, indígenas- hubiesen sido los más afectados. Tras un paro nacional que se extendió del 3 al 13 de octubre, las medidas económicas fueron retiradas. Para mayor información sobre las medidas, las acciones de los sectores vulnerados y las represalias del gobierno, se sugiere consultar las siguientes fuentes: Wambra, Pichincha Comunicaciones1, Crisis y CONAIE.

El propósito de este post es demostrar cómo ejecuté un análisis con los tweets generados entre el 1 y el 10 de octubre, considerando tres hashtags populares durante el paro nacional en Ecuador. La premisa es que la elección de un hashtag expone la intención del usuario, de reportar los hechos acontecidos durante el paro o de desaprobar al mismo. Originalmente trabajé con el paquete twitteR el cual permite obtener tweets a través de la API gratuita de Twitter; no obstante, es necesario ejecutar previamente una función de autenticación2.

library(twitteR)

setup_twitter_oauth("tus", "claves", "de", "consumidor")

wambra = userTimeline("wambraEc", 100, sinceID = 1212053690737135616,
                      excludeReplies = FALSE, includeRts = TRUE)

Si bien el paquete implementado ya posee una función que tabula los metadatos, preferí escribir mi propia función ajustada al análisis:

library(dplyr)
library(purrr)

user2na = function(user) ifelse(length(user) == 0, NA_character_, paste0("@", user))

coor2na = function(coor) ifelse(length(coor) == 0, NA_real_, as.double(coor))

list2tbl = function(twts_list, time_zone = "UCT") tibble(
   isrt = map_lgl(twts_list, ~.$isRetweet),
   rts  = map_dbl(twts_list, ~.$retweetCount),
   favs = map_dbl(twts_list, ~.$favoriteCount),
   time = map_dbl(twts_list, ~.$created) %>%
      as.POSIXct(tz = time_zone, origin = "1970-01-01"),
   user = map_chr(twts_list, ~user2na(.$screenName)),
   rpto = map_chr(twts_list, ~user2na(.$replyToSN)),
   text = map_chr(twts_list, ~.$text),
   lon  = map_dbl(twts_list, ~coor2na(.$longitude)),
   lat  = map_dbl(twts_list, ~coor2na(.$latitude)))

list2tbl(wambra, "EST") %>% tail(3)

Una vez tabulados los metadatos, existe una variedad de aspectos que pueden ser analizados. Un par de ideas se derivan de este conteo. La inclusión de #EcuadorPaisDePaz se relaciona con un desprecio por las protestas, pero también se relaciona con el porcentaje más bajo de tweets únicos y la razón más alta de tweets por cuenta. Aunque los números no son atípicos, estas son las características del spam. En contraposición, el uso de #ParoNacionalEcuador demuestra el porcentaje más alto de publicaciones útiles, y la tasa más baja de contribución por usuario; este hashtag fue comúnmente utilizado para reportar con fotos y videos las situaciones violentas y la represión ejecutada por los aparatos del estado.

Además, al contar el uso por cuenta, se halló que solo tres cuentas superaron en 85 el uso de #ParoNacionalEcuador; una de ellas fue la de Pichincha Comunicaciones. Por el contrario, fueron catorce las cuentas que usaron #EcuadorPaisDePaz más de 85 veces, entre ellas: los ministerios de transporte y obras públicas, de desarrollo urbano y vivienda, y de salud pública; las secretarias técnica, del deporte, y de comunicación de la presidencia; el servicio de rentas internas y varias cuentas zonales del ministerio de inclusión económica y social. Este hecho confirma que se trata de propaganda mediática por parte de las instituciones públicas, para desmerecer las demandas de los sectores movilizados.

Los datos que alimentaron la tabla de conteo fueron obtenidos con tres ejecuciones de la siguiente función, variando cada vez el hashtag de interés. Notar el uso de un filtro que excluye los retweets, pues de otra manera los datos devueltos excederían el límite impuesto de veinte mil.

searchTwitter("#ParoNacionalEcuador -filter:retweets", 2e4,
              since = "2019-10-01", until = "2019-10-10")

Desafortunadamente volver a ejecutar esta función no devolverá un solo tweet. Desconocía que la versión gratuita de la API solo recupera datos con una semana3 de antigüedad como máximo, a diferencia del sitio web que puede acceder a cualquier época. Es una lástima, pues no conservé los datos cuando ejecuté el análisis en octubre. Con todo existe una alternativa; básicamente, es un programa en java que simula la navegación en el sitio web de Twitter y recupera los datos conforme son presentados. Los siguientes comandos representan la obtención de tweets que contienen #ParoNacionalEcuador a través de la command line.

PS C:\> git clone https://github.com/Jefferson-Henrique/GetOldTweets-java.git
PS C:\> cd GetOldTweets-java-master
PS C:\GetOldTweets-java-master> java -jar got.jar querysearch="#ParoNacionalEC" since=2019-10-01 until=2019-10-15

Los resultados son escritos en un archivo de valores separados por comas -punto y coma en realidad- para el cual escribí otra función, observando también que ocurría un uso inadecuado de las comillas:

read.twts = function(csv){
   
   lines = gsub("\";", "';", gsub(";\"", ";'", readLines(csv)))
   
   cols = c(username = "NULL", date = "POSIXct", text = "character", geo = "NULL")
   
   df = read.csv2(text = lines, colClasses = cols, na.strings = "", quote = "'")
   
   df$user = gsub("^https://twitter.com/(\\w+)/status/\\d+$", "@\\1", df$permalink)
   
   df$id = as.numeric(df$id)
   
   tibble::as_tibble(df)}

Finalmente para leer los tweets recuperados: read.twts("output_got.csv")

En este caso se han recuperado solamente 244 de los 11471 tweets que originalmente se obtuvieron vía API. Es una cantidad mínima, pero es la alternativa a contratar un producto comercial de Twitter; a continuación, el desafío es averiguar hasta que nivel el análisis es replicable con esta cantidad reducida de datos.

  1. A la fecha de publicación, el sitio web se encuentra suspendido debido a denuncias de la presidencia. De cualquier forma Pichincha Comunicaciones, al igual que las otras fuentes sugeridas, presenta en su Twitter la cobertura que realizó durante el paro.↩

  2. La autenticación consiste en suministrar unas claves personales de consumidor y de acceso; para obtener esas claves uno debe poseer primero una cuenta de desarrollador.↩

  3. El ejemplo con tweets de Wambra sí funciona porque utiliza una API diferente; una que accede a cualquier época, pero en una única cuenta. La búsqueda por hashtag o texto sí está restringida a una semana.↩
